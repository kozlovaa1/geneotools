# Project Roadmap

> GeneoTools — браузерный инструмент для локального просмотра, анализа, редактирования и обратной сборки генеалогических баз "Древо Жизни 6" (`.atdb`).

## Milestones

- [x] **Стабилизация качества MVP** — довести lint/build до стабильного green, убрать остаточный debug-шум и зафиксировать минимальный smoke-check для основного сценария.
- [x] **Декомпозиция `.atdb`-процессора** — разделить монолитный `lib/sqlProcessor.ts` на модули чтения метаданных, персон, родов, событий, мест и общих SQL-хелперов.
- [x] **Исследование тестовой базы ATDB** — на эталонной тестовой базе уточнить фактическую схему таблиц, `Fields`, `ValuesStr`, `ValuesDates`, `ValuesLinks`, `EventDetails` и связи между записями.
- [x] **Эксперименты по неопределённым ATDB-сущностям** — после базового исследования и проверки `yaman-test-full.atdb` провести controlled diff-эксперименты в «Древо Жизни 6» для `rec_table=8`, `18`, `21`, пользовательских ролей событий (`rec_table=6`), «Полей данных» (`rec_table=10`) и других полей со статусом `unknown` / `needs more samples`.
- [x] **Проверка схемы на нескольких ATDB-fixtures** — прогнать безопасный schema-check на других разрешенных тестовых `.atdb` файлах, сравнить `rec_table`, `Fields`, `EventTypes`, `EventRoles`, `Values*` и parse/build counts с `yaman-test.atdb`, не публикуя персональные значения.
- [x] **Универсальный mapping и документация формата ATDB** — на основе нескольких проверенных fixtures выделить общий справочник формата `.atdb` без привязки к конкретной тестовой базе и вынести интерпретацию `rec_table`, `Fields`, ролей событий и `Values*` в явный слой правил с разделением invariant-правил, fixture-specific observations и legacy fallback; multi-fixture schema/docs сохранить как compatibility evidence и regression-контур, а не как конкурирующий канон формата.
- [x] **Устранение parse-build drift** — закрепить hard-fail regression gate для `parse -> build -> reparse`, чтобы любое ненулевое изменение counts по персонам, родам, событиям или местам блокировало smoke/matrix проверку; исторические увеличения `665 -> 678` и `7586 -> 8116` остаются только контекстом прошлых дефектов.
- [x] **Надежная обратная сборка `.atdb`** — расширить экспорт от сохранения исходной базы к контролируемому применению пользовательских изменений с валидацией и защитой структуры SQLite.
- [x] **Редактирование данных в UI** — добавить безопасное редактирование ключевых полей персон, родов, событий и мест с локальным состоянием изменений до скачивания файла.
- [ ] **Массовое редактирование полей** — добавить массовое заполнение выбранных полей и массовую замену строк по фильтрам/выборке с предпросмотром изменений перед применением.
- [ ] **Фильтрация и поиск по данным** — добавить быстрый поиск, фильтры и удобную навигацию по большим генеалогическим базам.
- [ ] **Упрощение табличного интерфейса** — разделить общий `DataTable` на специализированные таблицы, вынести сортировку и улучшить читаемость больших наборов данных.
- [ ] **Тестовый контур для парсинга и сборки** — добавить unit/smoke-тесты для ключевых маппингов `.atdb`, родительских связей, дат и сценария parse-build.
- [ ] **Поддержка вариативных схем ATDB** — накопить набор проверенных схем/примеров и документировать ограничения совместимости без раскрытия пользовательских данных.
- [ ] **Производительность больших баз** — оптимизировать обработку и рендеринг баз с тысячами записей, включая виртуализацию таблиц при необходимости.
- [ ] **Этап документации** — синхронизировать README/DOCS, `docs/atdb_format.md` и документацию схемы маппинга с фактической архитектурой, ограничениями MVP и безопасной локальной моделью обработки данных.
- [ ] **Стандартизация среды Node.js 22** — зафиксировать Node.js 22 LTS и совместимую версию npm в `package.json` и `.nvmrc`, синхронизировать требования документации, Docker и CI, не влияя на версии Node.js других локальных проектов.
- [ ] **Поставочный контур** — закрепить CI/CD, Docker/production-конфигурацию и проверяемый release-flow для развертывания.
- [x] **Базовый MVP загрузки и просмотра `.atdb`** — реализованы загрузка файла, проверка SQLite, локальный парсинг через `sql.js` и табличный просмотр персон, родов, событий и мест.
- [x] **Базовый экспорт `.atdb`** — реализован пользовательский сценарий скачивания обновленного `.atdb` на основе текущих данных и исходного буфера.
- [x] **AI Factory контекст и архитектурная документация** — добавлены описание проекта, архитектурные правила, AGENTS.md и документы анализа/refactoring-plan.
