# Project Roadmap

> GeneoTools — браузерный инструмент для локального просмотра, анализа, редактирования и обратной сборки генеалогических баз "Древо Жизни 6" (`.atdb`).

## Milestones

- [x] **Стабилизация качества MVP** — довести lint/build до стабильного green, убрать остаточный debug-шум и зафиксировать минимальный smoke-check для основного сценария.
- [x] **Декомпозиция `.atdb`-процессора** — разделить монолитный `lib/sqlProcessor.ts` на модули чтения метаданных, персон, родов, событий, мест и общих SQL-хелперов.
- [ ] **Исследование тестовой базы ATDB** — на эталонной тестовой базе уточнить фактическую схему таблиц, `Fields`, `ValuesStr`, `ValuesDates`, `ValuesLinks`, `EventDetails` и связи между записями.
- [ ] **Централизованный mapping формата ATDB** — на основе исследования тестовой базы вынести интерпретацию полей и ролей в явный слой правил с fallback-поведением.
- [ ] **Надежная обратная сборка `.atdb`** — расширить экспорт от сохранения исходной базы к контролируемому применению пользовательских изменений с валидацией и защитой структуры SQLite.
- [ ] **Редактирование данных в UI** — добавить безопасное редактирование ключевых полей персон, родов, событий и мест с локальным состоянием изменений до скачивания файла.
- [ ] **Массовое редактирование полей** — добавить массовое заполнение выбранных полей и массовую замену строк по фильтрам/выборке с предпросмотром изменений перед применением.
- [ ] **Фильтрация и поиск по данным** — добавить быстрый поиск, фильтры и удобную навигацию по большим генеалогическим базам.
- [ ] **Упрощение табличного интерфейса** — разделить общий `DataTable` на специализированные таблицы, вынести сортировку и улучшить читаемость больших наборов данных.
- [ ] **Тестовый контур для парсинга и сборки** — добавить unit/smoke-тесты для ключевых маппингов `.atdb`, родительских связей, дат и сценария parse-build.
- [ ] **Поддержка вариативных схем ATDB** — накопить набор проверенных схем/примеров и документировать ограничения совместимости без раскрытия пользовательских данных.
- [ ] **Производительность больших баз** — оптимизировать обработку и рендеринг баз с тысячами записей, включая виртуализацию таблиц при необходимости.
- [ ] **Этап документации** — синхронизировать README/DOCS, `docs/atdb_format.md` и документацию схемы маппинга с фактической архитектурой, ограничениями MVP и безопасной локальной моделью обработки данных.
- [ ] **Поставочный контур** — закрепить CI/CD, Docker/production-конфигурацию и проверяемый release-flow для развертывания.
- [x] **Базовый MVP загрузки и просмотра `.atdb`** — реализованы загрузка файла, проверка SQLite, локальный парсинг через `sql.js` и табличный просмотр персон, родов, событий и мест.
- [x] **Базовый экспорт `.atdb`** — реализован пользовательский сценарий скачивания обновленного `.atdb` на основе текущих данных и исходного буфера.
- [x] **AI Factory контекст и архитектурная документация** — добавлены описание проекта, архитектурные правила, AGENTS.md и документы анализа/refactoring-plan.
