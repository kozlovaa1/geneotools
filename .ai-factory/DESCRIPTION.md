# Проект: GeneoTools

## Обзор

**GeneoTools** — браузерный инструмент для просмотра, анализа, фильтрации, редактирования и обратной сборки генеалогических баз программы «Древо Жизни 6» (`.atdb`).

Файлы `.atdb` обрабатываются локально в браузере через `sql.js`: пользователь загружает SQLite-базу, приложение извлекает данные в типизированную структуру, показывает их в таблицах и позволяет выгрузить обновленный `.atdb` файл.

## Основные возможности

- Загрузка локального `.atdb` файла через drag-and-drop.
- Проверка SQLite-заголовка и открытие базы через `sql.js`.
- Извлечение персон, родов, событий, мест и метаданных.
- Восстановление родительских связей из событий рождения (`EventDetails.er_id`).
- Извлечение строковых значений из `ValuesStr`.
- Отображение данных в таблицах с сортировкой.
- Экспорт текущих данных обратно в `.atdb` на основе исходного буфера.
- Работа без внешних API и без сохранения пользовательских данных на сервере.
- Regression-проверки mapping, table-view, draft, batch edit, write-safety и strict rebuild contract.
- Контейнерный запуск через `Dockerfile` и `docker-compose.yml`.

## Tech Stack

- **Язык:** TypeScript 5
- **Framework:** Next.js 16 (App Router)
- **UI:** React 19, Tailwind CSS 4, Lucide React
- **SQLite:** `sql.js` для обработки `.atdb` в браузере
- **ORM:** отсутствует, используются прямые SQL-запросы через `sql.js`
- **Runtime:** Node.js 20 для сборки и Next.js runtime

## Архитектурные заметки

- `app/` отвечает за Next.js App Router и верхнеуровневый пользовательский сценарий.
- `components/` содержит UI-компоненты и таблицы; SQL-логика должна оставаться вне компонентов.
- `lib/` содержит доменную и инфраструктурную логику обработки `.atdb`.
- `scripts/` содержит вспомогательные проверки и сборочные скрипты.
- `docs/` содержит публичную пользовательскую и техническую документацию по формату `.atdb` и архитектуре.
- Данные пользовательских файлов должны оставаться локальными для браузерной сессии.
- Подробные архитектурные правила описаны в `.ai-factory/ARCHITECTURE.md`.
- **Паттерн:** Layered Modular App

## Нефункциональные требования

- **Производительность:** поддерживать обработку баз с тысячами записей без заметных зависаний UI.
- **Безопасность:** не отправлять `.atdb` файлы и персональные данные во внешние сервисы.
- **Типобезопасность:** сохранять строгий TypeScript и общие модели в `lib/types.ts`.
- **Совместимость:** учитывать вариативность схем `.atdb` и необязательность отдельных таблиц/полей.
- **Развертывание:** оставаться совместимым с обычной сборкой Next.js.
