# Fail fast for unsafe ATDB fixture verification inputs

**Date:** 2026-06-02 15:05
**Files:** scripts/inspect-atdb-schema.mjs, scripts/check-atdb-fixtures.mjs, docs/atdb_schema_yaman.snapshot.json
**Severity:** high

## Problem

Verification found that `scripts/inspect-atdb-schema.mjs` could accept a manual positional input such as `package.json --check`, read it, print fixture metadata, and then fail only after attempting SQLite parsing. The same path could also inherit the tracked Yaman snapshot output selection. The multi-fixture redaction gate also did not fully enforce the planned public artifact marker contract.

## Root Cause

Fixture input resolution still defaulted to the Yaman registry entry when no explicit fixture label was provided, even if a manual positional path existed. The guard logic also exempted `--check`, assuming no snapshot write meant no risk, but the script still read potentially private input and emitted derived metadata. Redaction validation was split between a narrow content scan and a partial snapshot exclusion check, so the code did not encode the full redaction contract.

## Solution

Added explicit manual-input detection and fail-fast output selection validation before any fixture existence check or file read. Manual non-default inputs now require explicit `--output` or `ATDB_SCHEMA_OUTPUT` unless the command is only validating an existing snapshot. Added `[FIX:atdb-schema-output]` logging for blocked unsafe invocations. Expanded the tracked snapshot `safety.excludes` contract and made `check-atdb-fixtures.mjs` verify all required exclusion categories plus concrete raw leak patterns such as UUIDs, user home paths, and local-only snapshot paths.

## Prevention

- Treat read-side metadata from private fixtures as sensitive, not only written snapshots.
- Apply fixture safety guards before `fs.existsSync`, `fs.readFileSync`, SQLite initialization, or logging fixture bytes.
- Keep public redaction gates split between contract checks for allowed/excluded categories and concrete regex checks for raw leaks.
- Add negative CLI repro checks for unsafe manual fixture input when scripts support both registry labels and positional paths.
