# Multi-fixture ATDB gate safety fixes

**Date:** 2026-06-02 16:36
**Files:** scripts/inspect-atdb-schema.mjs, scripts/check-atdb-fixtures.mjs, scripts/check-atdb-fixtures-regression.mjs, package.json, docs/atdb_schema_yaman.snapshot.json
**Severity:** high

## Problem

The staged multi-fixture ATDB schema changes still had three blocking review findings:

- manual `.atdb` input could fall back to the tracked yaman snapshot output without an explicit `--output`
- local-only fixtures could make the matrix wrapper fail when their snapshots were missing
- the redaction gate only enforced part of the declared snapshot exclusion contract

## Root Cause

The fixture registry introduced tracked and local-only fixture metadata, but the wrapper still assumed every fixture had a readable snapshot after schema mode. The inspect script also resolved the default tracked fixture before checking whether the caller supplied a manual input path, which made manual paths inherit the tracked output path.

The redaction check used marker matching that only effectively enforced `ValuesStr.vstr`, leaving the rest of the declared exclusion list as documentation rather than a hard gate.

## Solution

Added a pre-read output selection guard for manual fixture paths, requiring `--output` or `ATDB_SCHEMA_OUTPUT` unless the manual path is the tracked default fixture.

Extended the multi-fixture wrapper with fixture/snapshot availability checks. Missing local-only fixture inputs now produce safe `warning`/`skipped` rows, while missing tracked baseline artifacts still fail.

Expanded the tracked snapshot exclusions and made the redaction gate check the full required exclusion set plus safer public-content patterns.

Added a dependency-free regression script that runs the fixture diff wrapper from a temporary project root with only the tracked baseline snapshot present. It asserts that missing local-only fixtures are skipped and the wrapper still exits successfully.

## Prevention

- Review scripts that support both registry labels and arbitrary input paths for default-output fallback risks.
- Treat local-only fixture absence as a first-class matrix state instead of reading generated files unconditionally.
- Redaction contracts should be executable checks over exact required markers, not prose-only policy.
- Keep fixture-free regression commands for any safety behavior that depends on local-only files being absent.
