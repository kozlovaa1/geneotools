# Simple ValuesDates with NULL secondary columns

**Date:** 2026-07-02 17:25
**Files:** `lib/atdb/dates.ts`, `scripts/check-atdb-dates.mjs`, `scripts/check-atdb-parser-contract.mjs`
**Severity:** high

## Problem

Обычные ATDB-даты с `NULL` в колонках `y2/m2/d2` могли считаться non-simple. Из-за этого UI оставлял поддержанные даты рождения и смерти read-only, а explicit export мог получить `preflight.non_simple_date_edit`.

## Root Cause

Код проверял наличие вторичной даты через `value !== undefined`. Для `sql.js` реальные SQL `NULL` приходят как `null`, а не как отсутствие свойства. Поэтому строки `ValuesDates` с полной optional-схемой и пустыми secondary columns ошибочно выглядели как содержащие вторичную дату.

## Solution

`isSimpleAtdbDateMetadata()` теперь считает secondary date присутствующей только когда хотя бы одна из частей `y2/m2/d2` не равна `null` и не равна `undefined`. Добавлены regression cases:

- direct date-helper case для `{ y2: null, m2: null, d2: null, type: 0 }`
- parser-contract assertions, что synthetic SQL rows с `NULL` secondary columns читаются как `isSimple=true`

Runtime logging не добавлялся, потому что ATDB parser уже имеет redacted diagnostics, а дополнительный per-row debug log для нормального simple path создал бы шум без диагностической ценности.

## Prevention

- При чтении `sql.js` rows различать `undefined` как отсутствующую колонку и `null` как SQL NULL.
- Для optional ATDB metadata добавлять regression на обе формы: minimal schema без колонки и full schema с `NULL`.
- Review date helpers должен проверять не только display/value, но и `isSimple`, потому что именно этот флаг управляет write-safe UI и preflight.
