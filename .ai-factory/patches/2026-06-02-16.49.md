# Read-only ATDB schema check blocked manual fixtures

**Date:** 2026-06-02 16:49
**Files:** scripts/inspect-atdb-schema.mjs, scripts/check-atdb-fixtures-regression.mjs
**Severity:** medium

## Problem

`node scripts/inspect-atdb-schema.mjs <manual-fixture> --check` failed with
`manual fixture input requires --output or ATDB_SCHEMA_OUTPUT`.

The command is read-only, so it should not require an explicit snapshot output and should be able to report a missing manual fixture as `status: skipped`.

## Root Cause

The new output selection guard protected tracked snapshots from accidental overwrite by manual fixture input, but it only exempted `--snapshot-check`.

`--check` uses the same no-write behavior, but it still passed through the manual-output guard. That mixed write-safety policy with read-only validation behavior.

## Solution

Updated `assertSafeOutputSelection()` to return early for `checkOnly`.

Extended the fixture regression script with a manual missing fixture check that runs `inspect-atdb-schema.mjs <manual> --check` without `--output`, asserts exit code `0`, verifies `status: skipped`, and ensures the output-requirement error is absent.

## Prevention

- Treat all read-only CLI modes as explicit exceptions to output-overwrite guards.
- Add regression coverage for both write-protection failures and read-only bypass behavior.
- When adding safety guards, test at least one missing-input path before any sensitive file is read or written.
