# Remove raw ATDB values from public format documentation

**Date:** 2026-06-01 13:09
**Files:** docs/atdb_format.md
**Severity:** high

## Problem

The verification gate found real `ValuesStr` examples and death-cause text in the tracked `docs/atdb_format.md` overview. This violated the ATDB research plan safety requirement that public artifacts must not include personal names, places, notes, causes of death, document paths, GUIDs, source text, or raw string values from the fixture.

## Root Cause

The overview document was updated from validated fixture knowledge, but the redaction rule was applied to the schema artifact more strictly than to the older summary document. The summary kept illustrative examples even though the plan required structural evidence only.

## Solution

Replaced raw examples in `docs/atdb_format.md` with structural wording. The document now states that real values are intentionally omitted and points readers to `f_id`, `rec_table`, counts, and the redacted snapshot.

## Prevention

- Treat all tracked ATDB documentation as public unless explicitly marked local/private.
- During review, search changed docs for known fixture strings and value examples before commit.
- Prefer field IDs, table names, counts, and confidence labels over illustrative values from `.atdb` records.
