# Guard alternate ATDB schema snapshot output

**Date:** 2026-06-01 13:53
**Files:** scripts/inspect-atdb-schema.mjs, README.md
**Severity:** high

## Problem

`scripts/inspect-atdb-schema.mjs` allowed `ATDB_SCHEMA_FIXTURE` or a positional path to inspect any local `.atdb` fixture, but still defaulted snapshot output to the tracked `docs/atdb_schema_yaman.snapshot.json`. Running `npm run schema:atdb` with a private alternate fixture could silently overwrite the public Yaman snapshot with local aggregate data.

## Root Cause

The script separated fixture selection from output selection but did not bind the default tracked output path to the default tracked Yaman fixture. The CLI supported flexible fixture input, while the output default assumed the canonical research fixture.

## Solution

Added path normalization to detect whether the selected fixture is the default `yaman-test.atdb`. For non-default fixtures, non-check runs now require explicit `ATDB_SCHEMA_OUTPUT` or `--output`; otherwise the script exits before reading the file and logs a safe `[FIX:atdb-schema-output]` message. README now documents the required explicit output for alternate fixtures.

## Prevention

Review any script that accepts private/local input and writes tracked artifacts. If input is configurable but output is tracked by default, require explicit output for non-canonical inputs or route output to ignored local paths.
