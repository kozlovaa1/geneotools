# Regenerate tracked ATDB snapshot after artifact contract change

**Date:** 2026-06-01 21:17
**Files:** docs/atdb_schema_yaman.snapshot.json
**Severity:** medium

## Problem

`npm run schema:atdb:snapshot:check` failed with `schema snapshot missing required section: globalMeta`. Because `schema:atdb:diff:check` chains the snapshot check first, the new diff gate failed as well.

## Root Cause

`scripts/inspect-atdb-schema.mjs` was updated to require the `globalMeta` section in versioned public snapshots, but the tracked `docs/atdb_schema_yaman.snapshot.json` artifact had not been regenerated with the same contract. The code and artifact schema drifted apart.

## Solution

Regenerated the canonical redacted Yaman snapshot with `npm run schema:atdb`. The tracked snapshot now includes `artifactVersion`, `redactionPolicy`, the full `sections` list, and sanitized `globalMeta` values with hashes and lengths instead of raw GUID or parameter payloads.

## Prevention

- After changing required snapshot sections, regenerate and validate all tracked snapshot artifacts in the same patch.
- Keep `schema:atdb:snapshot:check` in the verification gate for any artifact contract change.
- Review generated snapshots for raw `ValuesStr`, GUIDs, paths, notes, source text, and personal values before accepting them.
