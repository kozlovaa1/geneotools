# Начало работы

## Что это

GeneoTools — клиентское Next.js приложение для локальной работы с `.atdb` файлами. Пользователь загружает базу в браузере, приложение читает SQLite-структуру через `sql.js`, показывает данные в таблицах и собирает новый файл для скачивания.

## Требования

- Node.js 20+
- npm
- Современный браузер с поддержкой `ArrayBuffer`, `Blob` и `FileReader`

## Установка

```bash
npm install
```

## Запуск

```bash
npm run dev
```

После запуска откройте `http://localhost:3000`.

## Основной сценарий

1. Загрузите локальную копию `.atdb` через drag-and-drop или file picker.
2. Приложение покажет фазу чтения файла, затем проверит SQLite header и откроет базу через `sql.js`.
3. После парсинга будут доступны вкладки «Персоны», «Роды», «События» и «Места».
4. Используйте быстрый поиск, фильтр по одному полю или сортировку в активной вкладке; при тяжёлом обновлении таблица покажет компактный статус.
5. Редактируйте только разрешённые поля:
   - персоны: фамилия, имя, отчество, пол, существующие ссылки на место рождения и смерти;
   - роды: название рода, мужская фамилия, женская фамилия, комментарий и цвет;
   - места: название, краткое название и комментарий.
6. Для массового редактирования выберите строки, откройте «Массовое редактирование», задайте поле, область и операцию, затем сначала постройте предпросмотр.
7. Применение массового редактирования меняет только локальный draft.
8. Счётчик рядом с кнопкой скачивания показывает количество изменённых полей и записей.
9. Пока изменений нет, кнопка скачивания обновлённого файла отключена.
10. Скачайте обновлённый `.atdb`; во время подготовки повторный экспорт отключён, исходный локальный файл не перезаписывается.

## Массовое редактирование

Доступные области:

| Область | Что меняется |
|---------|--------------|
| Выбранные строки | Только явно выбранные строки активной editable вкладки |
| Все строки вкладки | Все строки текущей editable вкладки |
| Строки по условию | Строки, прошедшие `contains`, `equals`, `empty` или `not empty` |

Доступные операции:

| Операция | Поведение |
|----------|-----------|
| Заполнить | Записывает выбранное значение в write-safe поле |
| Очистить | Очищает write-safe значение по правилам strict rebuild |
| Заменить строку | Работает только для строковых write-safe полей |

Предпросмотр показывает affected, skipped и no-op counts, reason codes, изменения по ID и статус актуальности. Кнопка применения активна только для актуального предпросмотра; во время подготовки предпросмотра или применения повторное действие отключено.

## Полезные команды

```bash
npm run lint
npx tsc --noEmit
npm run build
npm run mapping:atdb:check
npm run test:atdb:table-view
npm run test:atdb:table-components
npm run test:atdb:edit-draft
npm run test:atdb:batch-edit
npm run test:atdb:write-safety
npm run test:atdb:rebuild-contract
```

| Команда | Что проверяет |
|---------|---------------|
| `npm run lint` | ESLint для приложения и скриптов |
| `npx tsc --noEmit` | Типы без генерации сборочных файлов |
| `npm run build` | Production-сборку Next.js |
| `npm run mapping:atdb:check` | Канонический `lib/atdb/mapping.json` и отсутствие конфликтующих кодов |
| `npm run test:atdb:table-view` | Поиск, фильтры, сортировку, visible IDs и labels |
| `npm run test:atdb:table-components` | Контракты таблиц, selection lookup, pending/status surfaces и dialog accessibility |
| `npm run test:atdb:edit-draft` | Локальный draft state и сборку `AtdbChangeSet` |
| `npm run test:atdb:batch-edit` | Предпросмотр и применение массового редактирования |
| `npm run test:atdb:write-safety` | Защиту неизвестных `Values*` и недопустимых write paths |
| `npm run test:atdb:rebuild-contract` | Success/failure paths strict rebuild API |

## Что проверить вручную

- `.atdb` файл открывается без отправки данных на сервер.
- Во время чтения и парсинга загрузчик заблокирован и показывает текущую фазу.
- Вкладки переключаются без потери локального draft.
- Поиск, фильтр, очистка фильтра и сортировка работают в активной вкладке; при обновлении появляется компактный статус.
- Checkbox «выбрать все» выбирает только текущие видимые строки.
- Массовое редактирование требует актуальный предпросмотр и применяет изменения только в draft.
- Экспорт доступен только при ненулевом `AtdbChangeSet`.
- Во время подготовки экспорта кнопка скачивания не запускает повторную сборку.
- Скачанный файл открывается в ожидаемом инструменте без изменения исходной локальной копии.

## Ограничения

- Создание и удаление записей не поддерживаются.
- События, даты, участники событий, родственные связи, notes/occupation, metadata, `Global`, `Fields`, `Recs`, `EventRoles` и custom fields намеренно read-only.
- Ссылки `birthPlaceId` и `deathPlaceId` редактируются только там, где исходный parse уже нашёл существующую ссылку на место.
- Глобальный поиск по всем вкладкам, сложный конструктор условий и виртуализация больших таблиц остаются отдельными этапами.
